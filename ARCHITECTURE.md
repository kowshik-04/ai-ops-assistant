### ğŸ§  Planner Agent
**File**: `agents/planner.py`
- **Input**: Natural language query
- **Process**: Uses LLM to decompose task into steps
- **Output**: Structured JSON plan with tool selections
- **LLM Usage**: JSON mode for guaranteed valid output

### âš™ï¸ Executor Agent
**File**: `agents/executor.py`
- **Input**: Structured plan from Planner
- **Process**: Executes each step sequentially, calling appropriate tools
- **Output**: Results from all API calls + execution log
- **Error Handling**: Captures failures, continues execution

### âœ… Verifier Agent
**File**: `agents/verifier.py`
- **Input**: Original query + execution results + log
- **Process**: 
  1. Validates data completeness
  2. Uses LLM to generate human-readable summary
- **Output**: Final structured response with validation status
- **LLM Usage**: Quality checks and summary generation

## Key Design Principles

1. **Separation of Concerns**
   - Each agent has ONE responsibility
   - Tools are isolated and reusable
   - Clear boundaries between layers

2. **Resilience**
   - Retry logic for transient failures
   - Graceful degradation
   - Detailed error logging

3. **Type Safety**
   - Pydantic schemas for validation
   - Type hints throughout
   - Structured JSON outputs

4. **Extensibility**
   - Easy to add new tools
   - Agent interface is consistent
   - LLM prompts are modular

5. **Production Ready**
   - Proper error handling
   - Logging and observability
   - One-command deployment

---

**This architecture ensures reliability, maintainability, and scalability.**
